What is CI/CD?

CI/CD Stand for Continuous Integration and Continuous Deployment, it bridges the gap between development operation activities and teams by allowing automation in the building, testing, and deployment of applications. Continuous Delivery/Deployment allows a developer's changed to an application to be automatically bug tested and uploaded to a repository, where they then can be deployed to a live production environment such as Heroku.

Github CI/CD

When commits are pushed onto Github, CI/CD allows further instruction with that to do with that code once it reaches the repository. Testing is a very important workflow aspect of this process, as it allows GitHub to run instructions (typically a .yml file) given by the developer to ensure that the code still runs following changes in a commit. Deployment is also done through this same process. Of course, services like Heroku require login details in order to deploy, but how do we allow Github to login for us without having our details out in the open in the source code? GitHub as a feature called action secrets, these are stored in Github's side and in their place within the yml files are variables representing the action secrets. For Docker Hub's case, this is a username and password, but in Heroku's case, we need to use an Authorization Token.
