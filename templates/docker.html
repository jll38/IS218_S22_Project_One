What is Docker?

Docker is a tool that allows developers to deploy their applications in a sandbox environment, utilizing OS-level linux virtualization. Docker utilizes containers to run these software applications, rather than virtual machines, with the main difference being in computational power and efficiency. Docker simplifies and accelerates your workflow, while giving developers the freedom to innovate with their choice of tools, application stacks, and deployment environments for each project.

Containers

A container is a standard software unit that encapsulates code and all of its dependencies. They are fast and reliable. A Docker container image is a small, standalone software package that contains everything needed to run a program, including code, runtime, system tools, system libraries, and settings.

Docker Commands

Docker utilizes the commandline for most, if not, all of it's functionalities, more specifically it is designed around the Linux commandline. However, it is possible to run these commands in Windows utilizing WSL and Powershell.

docker -version | Returns the currently installed version of docker
docker pull (image name) | Used to pull images from the Docker Repository/Docker Hub
docker run -it -d (image name) | Creates a container from an image
docker ps | Returns a list of all running containers
docker exec -it (container id) | Accesses the running container
docker kill (container id) | Kills the container immediately
docker commit (container id) (username/imagename) | Creates new image of an edited container
Ex:
"docker commit 23f78beaa2f3 jll38/is218_s22_project_one:master"
docker push (username/imagename) | Pushes an image to the Docker Hub Repository
docker images | Lists all locally stored docker images
docker rm (container id) | Deletes a stopped container.
docker build | Builds an image from a dockerfile
